# Проектне завдання 9: Пошуковий аналіз датасету за допомогою Jupyter Notebook

       Мета виконання завдання: Виконати первинний пошуковий аналіз датасету для бізнес-задачі у середовищі Jupyter Notebook, щоб зрозуміти структуру даних, виявити проблеми якості, оцінити розподіл ознак, знайти залежності між змінними, перевірити початкові гіпотези щодо даних і підготувати обґрунтовані висновки для наступних етапів очищення, побудови ознак і моделювання.

План виконання:
1. Загальний опис датасету:
          - кількість рядків;
          - кількість стовпців;
          - список основних атрибутів;
          - типи даних ознак;
          - попередню роль ознак: ідентифікатори, числові, категоріальні, часові, текстові, цільова змінна.

2. Завантаження і первинний огляд даних у Jupyter Notebook У ноутбуці потрібно:
          - імпортувати потрібні бібліотеки;
          - зчитати датасет;
          - вивести перші 5–10 записів;
          - переглянути назви колонок;
          - перевірити типи даних;
          - побудувати зведену інформацію про датафрейм.
Мінімально бажані команди/кроки: head(), shape, columns, info(), describe() для числових ознак, describe(include='all') або аналог для загального огляду.

3. Аналіз повноти та якості даних Потрібно дослідити:
          - наявність пропущених значень;
          - кількість дублікатів;
          - некоректні значення;
          - неузгоджені категорії;
          - аномально малі/великі значення;
          - потенційні проблеми форматування дат, чисел, тексту.
Потрібно подати:
          - таблицю пропусків по кожній ознаці;
          - короткий висновок, які колонки є найбільш проблемними;
          - оцінку, чи можна використовувати датасет без додаткового очищення;
          - перелік щонайменше 3 потенційних ризиків для подальшого моделювання.

4. Одновимірний аналіз ознак Для кожного ключового типу ознак виконати аналіз:
          - Для числових ознак: розподіл значень; мінімум, максимум, середнє, медіана; стандартне відхилення; квартилі; можливі викиди.
          - Для категоріальних ознак: кількість унікальних значень; частоти категорій; наявність рідкісних класів; дисбаланс категорій.
          - Для часових ознак: період покриття даних; інтенсивність записів у часі; сезонність або нерівномірність.
          - Бажані візуалізації: гістограми; boxplot; bar chart; line chart для часових даних.

5. Двовимірний та багатовимірний аналіз На цьому кроці треба знайти взаємозв’язки між ознаками. Потрібно дослідити:
          - кореляції між числовими ознаками;
          - взаємозв’язок ключових ознак із цільовою змінною;
          - залежності між категоріальними та числовими змінними;
          - залежності у часі;
          - ознаки, які можуть бути корисними для побудови нових features.
Можливі інструменти: correlation matrix; heatmap; scatterplot; groupby-аналіз; pivot-таблиці; порівняльні boxplot/barplot за класами.

6. Аналіз цільової змінної або бізнес-результату Якщо задача має цільову змінну, треба:
         - описати її зміст;
         - оцінити розподіл класів або значень;
         - перевірити наявність дисбалансу;
         - вказати, які наслідки це матиме для вибору метрик і алгоритмів.
Наприклад:
         - для класифікації — частка класів;
         - для регресії — діапазон і форма розподілу target;
         - для аномалій — приблизна частка підозрілих випадків;
         - для кластеризації — опис ознак, за якими потенційно буде проводитися групування.

7. Формування аналітичних висновків У кінці ноутбука потрібно сформувати текстові висновки:
         - які ознаки є найінформативнішими;
         - які ознаки треба трансформувати;
         - які колонки містять найбільше проблем;
         - які гіпотези підтверджені;
         - які попередні рішення можна прийняти для етапу очищення та побудови моделі.
Обов’язково дати відповідь на питання:
         1. Чи придатний датасет для вирішення бізнес-задачі?
         2. Які основні проблеми якості даних виявлено?
         3. Які ознаки потенційно найбільш корисні для моделі?
         4. Які кроки треба виконати на наступному етапі очищення?

Очікувані результати та критерії оцінювання:
   - (1 бал) Опис датасету та бізнес-контексту — чітко описана бізнес-тема, мета аналізу, джерело датасету, задачу машинного навчання, цільову змінну та основні атрибути.
   - (1 бал) Первинний технічний огляд даних — коректно виконано завантаження даних, показано структуру таблиці, типи даних, розмір датасету, первинну статистику.
   - (1 бал) Аналіз якості даних — виявлено пропуски, дублікати, аномальні значення, помилки форматів або інші дефекти даних; наведено аргументовані висновки.
   - (1 бал) Візуальний та статистичний аналіз ознак — побудовано достатню кількість графіків і таблиць; проаналізовано числові, категоріальні та, за наявності, часові ознаки.
   - (1 бал) Аналіз залежностей та цільової змінної — досліджено взаємозв’язки між ознаками та вплив ознак на цільову змінну або бізнес-показник; зроблено аналітичні висновки.
   - (1 бал) Якість оформлення та висновки — ноутбук логічно структурований, містить пояснення, підписи, проміжні та фінальні висновки; оформлення відповідає вимогам.
