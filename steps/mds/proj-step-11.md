# Проектне завдання 11: Побудова конвеєру для автоматичного опрацювання вхідних даних

      Мета виконання завдання: Навчитися проектувати, реалізовувати та тестувати ETL-конвеєр (Extract-Transform-Load) для автоматизованої обробки вхідних даних. Ознайомитися з практичними підходами до структуризації конвеєра, організації моніторингу, забезпечення надійності та перевірки ефективності роботи системи.

План виконання:
1. Аналіз вимог та планування
   - Сформулювати загальні вимоги до конвеєру:
          - Джерела даних (CSV, API, стрімінгові джерела)
          - Частота оновлення
          - Формати вихідних даних
          - Тип обробки: пакетна / потокова
   - Визначити середовище розгортання (Jupyter Notebook, локальна система або хмара).

2. Структуризація конвеєру
   - Розділити конвеєр на основні етапи:
          - Ingestion (витяг даних) – читання з джерела.
          - Validation (валідація) – перевірка структури та форматів.
          - Transformation (перетворення) – очищення, нормалізація, обчислення ознак.
          - Storage (збереження) – запис у базу даних або файл.
   - Побудувати схему (діаграму) із описом кожного етапу.
   - Використати Python (наприклад, Pandas, SQLAlchemy, Airflow, Prefect).

3. Налаштування автоматичної обробки
   - Реалізувати автоматичну обробку:
          - За допомогою планувальника (наприклад, cron або Prefect schedule)
          - Обробка при надходженні нового файлу (watcher)
   - Розробити скрипт Python із викликом ETL-конвеєру
   - Додати логування основних подій

4. Реалізація моніторингу
   - Визначити ключові метрики:
         - Час обробки
         - Обсяг оброблених даних
         - Кількість помилок / викинутих записів
   - Реалізувати просту систему логування або моніторингу:
         - Лог-файли / stdout
         - Використання Prometheus + Grafana (опціонально)
   - Побудувати схему моніторингу з поясненнями

5. Тестування ефективності обробки даних
   - Провести юніт-тестування окремих функцій конвеєру
   - Оцінити:
         - Швидкість обробки
         - Точність валідації
         - Стабільність при великому обсязі даних
   - Надати результати тестування у вигляді таблиці або графіка

Очікувані результати та критерії оцінювання:
   - (1 бал) Призначення конвеєру - чітко визначено бізнес-задачі, зазначено вхід та вихід конвеєру, пояснення конвеєра у проекті.
   - (1 бал) Структура конвеєру - схема з підписами для кожного етапу.
   - (1 бал) Схема моніторингу у конвеєрі - включно з логуванням та ключовими метриками
   - (1 бал) Розроблений скрипт - Python-код з усіма модулями
   - (1 бал) Результати тестування роботи конвеєру - таблиця або графік із результатами тестування обробки даних.
   - (1 бал) Якість оформлення та пояснення рішення - звіт оформлено акуратно, логічні пояснення, код та схеми читабельні й зрозумілі.
