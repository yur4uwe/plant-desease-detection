# Проектне завдання 12: Дослідження алгоритмів машинного навчання для вирішення задачі

      Мета виконання завдання: Дослідити можливі алгоритми машинного навчання (МН) для розв’язання обраної бізнес-задачі, порівняти їх за якістю, інтерпретованістю, складністю впровадження та відповідністю бізнес-вимогам, а також обґрунтовано вибрати базовий алгоритм для подальшої інтеграції в конвеєр даних і систему автоматизованого оцінювання.

План виконання:
1. Вибір алгоритмів для дослідження
   - Завантажте свій набір даних.
   - Для кожного типу задачі оберіть 3–5 моделей:
         - - Класифікація:
                - Logistic Regression
                - Decision Tree
                - Random Forest
                - Support Vector Machine
                - XGBoost / LightGBM
         - - Регресія:
                - Linear Regression
                - Random Forest Regressor
                - Gradient Boosting
                - Ridge / Lasso
         - - Кластеризація:
                - K-Means
                - DBSCAN
                - Gaussian Mixture Model
                - Agglomerative Clustering
         - - Аномалії:
                - Isolation Forest
                - One-Class SVM
                - AutoEncoder
         - Рекомендації:
                - Content-Based Filtering
                - Collaborative Filtering (Matrix Factorization)
                - Hybrid Models
         - Оптимізація:
                - Heuristic Algorithms (наприклад, Genetic Algorithm)
                - Linear Programming
                - Reinforcement Learning (спрощено)
   - Якщо немає бажаного алгоритму у списку, розгляньте свої альтернативи.

2. Навчання моделей та аналіз результатів
   - Розподіліть дані на три датасети: train, validation, test.
   - Побудуйте конвеєр (pipeline) для кожної моделі.
   - Обчисліть основні метрики:
          - Для класифікації: accuracy, precision, recall, F1-score
          - Для регресії: RMSE, MAE, R²
          - Для кластеризації: Silhouette Score, Davies-Bouldin Index
   - Візуалізуй результати:
          - Confusion Matrix, ROC-крива, регресійні графіки, кластерні розподіли.
3. Оцініть тенденцій: Data Drift / Concept Drift
   - Проімітуйте ситуацію зі зміною розподілу вхідних даних.
   - Визначіть:
          - Data Drift - зміна розподілу ознак (наприклад, вхідних параметрів)
          - Concept Drift - зміна залежності між ознаками та цільовою змінною
   - Використайте інструменти: evidently, alibi-detect або ж власні графіки розподілу.

4. Оцінювання та рейтинг моделей
   - Проведіть гіперпараметричну оптимізацію із використанням: GridSearchCV, RandomizedSearchCV, Optuna або Hyperopt
   - Проведіть додаткове налаштування 1–2 моделей.
   - Порівняти результати до і після оптимізації.
   - Створіть таблицю порівняння моделей (приклад наведено нижче)
   - Побудуйте фінальний рейтинг моделей.

Очікувані результати та критерії оцінювання:
   - (1 бал) Постановка задачі МН - коректно визначено тип МН-задачі.
   - (1 бал) Аналіз датасету для вибору алгоритму - описано характеристики даних, що впливають на вибір моделі; враховано типи ознак, дисбаланс, пропуски, часову структуру, обсяг даних.
   - (1 бал) Розроблені крипти - код у вигляді Jupyter Notebook або .py файлів.
   - (1 бал) Список обраних алгоритмів - таблиця з обраних алгоритмів, їх призначенням, коротким описом та списком гіпер параметрів для навчання чи інших параметрів.
   - (1 бал) Скріншоти візуалізацій - метрики, тенденції та інші візуалізовані показники.
   - (1 бал) Рейтингові таблиці моделей - таблиці порівняння та рейтингу моделей.

       
Разом: 6 балів
