# Проектне завдання 8: Побудова датасету для вирішення бізнес-задачі

      Мета виконання завдання: Сформувати підсумковий датасет для розв’язання обраної бізнес-задачі на основі зібраних і підготовлених даних. Датасет має бути структурованим, логічно обґрунтованим, придатним для подальшого пошукового аналізу, очищення, побудови конвеєра обробки даних та навчання моделі машинного навчання. Необхідно визначити цільову змінну, пояснювальні ознаки, структуру записів, джерела походження полів, а також перевірити відповідність датасету бізнес-меті проекту.

План виконання:
1. Визначення об’єкта спостереження згідно заданої теми Потрібно чітко визначити, що є одним записом датасету. Можливі приклади:
          - один клієнт;
          - одна транзакція;
          - один відгук;
          - один рейс;
          - один день роботи обладнання;
          - один музичний трек;
          - одна дорожня подія;
          - один студент.
Слід пояснити:
          - чому саме цей об’єкт обрано;
          - як він пов’язаний із бізнес-задачею;
          - чи немає змішування кількох рівнів гранулярності в одному датасеті.

2. Визначення цільової змінної Необхідно:
          - обрати цільову змінну (target);
          - пояснити, що саме вона означає в бізнес-контексті;
          - вказати тип цільової змінної: бінарна, багатокласова, числова, відсутня, якщо задача без учителя;
          - описати, як ця змінна формується.
Наприклад:
          - default_flag = 1, якщо клієнт не сплатив кредит у визначений строк;
          - churn = 1, якщо користувач скасував підписку;
          - review_label = fake / real;
          - yield_tons — прогнозована врожайність;
          - fraud_flag — шахрайська транзакція чи ні.
Якщо готової цільової змінної немає у джерелі, то потрібно описати правила її побудови.

3. Визначення ознак датасету Для кожної ознаки потрібно вказати:
            - назву поля;
            - короткий зміст;
            - тип даних;
            - одиницю вимірювання, якщо доречно;
            - джерело поля;
            - роль ознаки: ідентифікатор, числова, категоріальна, дата/час, текстова, похідна, цільова.
Рекомендується подати це у вигляді таблиці.

5. Формування структури датасету Необхідно описати:
            - формат датасету: CSV, Parquet, JSON, SQL-таблиця;
            - орієнтовну кількість записів;
            - орієнтовну кількість ознак;
            - наявність часової складової;
            - наявність текстів, зображень або зовнішніх довідників;
            - ключі зв’язку між таблицями, якщо датасет формується з кількох джерел.
Якщо використовується кілька таблиць, слід показати:
            - основну таблицю;
            - допоміжні таблиці;
            - правила об’єднання (join);
            - поле або набір полів для зв’язування.

6. Побудова датасету на основі ETL На цьому етапі студент повинен використати результати попередніх етапів, особливо збору даних та ETL-модуля. Потрібно:
          - отримати дані з відкритого джерела або кількох джерел;
          - виконати базову інтеграцію;
          - прибрати явно зайві поля;
          - залишити лише ті атрибути, які потенційно корисні для бізнес-задачі;
          - забезпечити єдині назви колонок;
          - уніфікувати формати дат, чисел, категорій;
          - сформувати файл або таблицю підсумкового датасету.

7. Первинна перевірка придатності датасету Необхідно виконати короткий контроль якості датасету:
          - чи є пропущені значення;
          - чи є дублікати;
          - чи є невалідні типи даних;
          - чи є аномальні або неможливі значення;
          - чи є зміщення цільового класу;
          - чи достатньо записів для навчання моделі;
          - чи відповідають поля бізнес-логіці. На цьому етапі не потрібно робити глибоке очищення — це буде деталізовано на наступних етапах.
Але базова перевірка обов’язкова.

8. Документування датасету Потрібно підготувати короткий паспорт датасету, де зазначити:
          - назву датасету;
          - бізнес-тему;
          - бізнес-задачу;
          - об’єкт спостереження;
          - цільову змінну;
          - список основних ознак;
          - джерело(а) даних;
          - дату формування;
          - формат збереження;
          - потенційні обмеження датасету.

Очікувані результати та критерії оцінювання:
   - (1 бал) Постановка бізнес-задачі — чітко сформульовано бізнес-проблему, правильно визначено тип задачі машинного навчання, пояснено, яку бізнес-користь дає майбутня модель.
   - (1 бал) Визначення об’єкта спостереження та цільової змінної — правильно визначено, що є одним записом, логічно обрано target, пояснено бізнес-зміст цільової змінної.
   - (1 бал) Опис джерел даних та логіки побудови датасету — вказано джерела, пояснено, як дані інтегруються, наведено логіку формування підсумкової таблиці.
   - (1 бал) Таблиця ознак датасету — наведено перелік полів, описано типи даних і ролі ознак, структура датасету зрозуміла та придатна до подальшого аналізу.
   - (1 бал) Якість побудованого датасету — датасет зібрано коректно, немає критичних структурних помилок, виконано базову перевірку пропусків, дублікатів і форматів.
   - (1 бал) Оформлення та обґрунтування — звіт оформлено акуратно, результати пояснені, зроблено висновок щодо придатності датасету для вирішення бізнес-задачі.
Разом: 6 балів
