# Проектне завдання 6: Вибір критеріїв для оцінювання якості моделі машинного навчання

      Мета виконання завдання: Обрати, обґрунтувати та формалізувати систему критеріїв оцінювання якості моделі машинного навчання відповідно до бізнес-задачі проекту, типу задачі машинного навчання, характеристик датасету та очікуваних результатів. Визначити набір метрик, правила їх інтерпретації, умови порівняння моделей та ризики помилкового оцінювання.

План виконання:
1. Визначення типу задачі машинного навчання Необхідно вказати, яку саме задачу вирішує модель: класифікація, регресія, кластеризація, ранжування, прогнозування часових рядів, рекомендаційна задача, коротко описати цільову змінну, пояснити, який результат вважається “якісним” з точки зору проекту. Результат: короткий опис ML-задачі та очікуваного типу виходу моделі.

2. Аналіз особливостей даних, що впливають на вибір метрик Необхідно проаналізувати:
           - баланс / дисбаланс класів;
           - наявність шуму та пропусків;
           - обсяг датасету;
           - кількість ознак;
           - типи ознак;
           - наявність часової залежності;
           - наявність викидів;
           - бізнес-наслідки помилок різного типу. Потрібно пояснити, чому не всі метрики підходять для вашого випадку.​ ​ Наприклад:
           - accuracy не є достатньою при сильному дисбалансі класів;
           - MSE може бути чутливою до великих викидів;
           - для бізнес-задачі важливішими можуть бути precision або recall, ніж загальна точність. Результат: перелік характеристик даних, які впливають на систему оцінювання.

3. Вибір основних метрик якості моделі Потрібно обрати основні метрики відповідно до типу задачі. Для класифікації можуть бути обрані: Accuracy, Precision, Recall, F1-score, ROC-AUC, PR-AUC, Log Loss, Confusion Matrix Для регресії: MAE, MSE, RMSE, R², MAPE​ ​ Для кластеризації: Silhouette Score, Davies–Bouldin Index, Calinski–Harabasz Index​ ​ Для часових рядів / прогнозування: MAE, RMSE, MAPE, SMAPE, WAPE​

      Для кожної метрики потрібно:
            - навести формальну назву;
            - коротко пояснити зміст;
            - описати, що означає краще / гірше значення;
            - пояснити, чому саме ця метрика релевантна проекту.​ Результат: таблиця або список із 3–5 основними метриками.

4. Вибір додаткових критеріїв оцінювання Окрім класичних метрик якості, слід оцінити додаткові критерії, якщо вони важливі для проекту:
            - інтерпретованість моделі;
            - швидкодія навчання;
            - швидкодія інференсу;
            - стійкість до шуму;
            - узагальнювальна здатність;
            - стабільність на різних підвибірках;
            - масштабованість;
            - можливість автоматичного моніторингу;
            - вартість помилки;
            - відповідність бізнес-обмеженням. Результат: 2–4 додаткові критерії з поясненням їх важливості.

5. Формування правил оцінювання та порівняння моделей Потрібно визначити:
          - як саме буде проводитися оцінювання: 
        - train/test split; 
        - cross-validation; 
        - time-series split;
          - які метрики є головними, а які допоміжними;
          - як буде прийматися рішення про вибір найкращої моделі;
          - які порогові значення вважаються прийнятними;
          - чи допускається компроміс між якістю, швидкістю та інтерпретованістю. Приклад:
          - основна метрика: F1-score;
          - допоміжні: precision, recall, ROC-AUC;
          - мінімально прийнятне значення F1-score: 0.78;
          - при близьких значеннях якості обирається модель з вищою інтерпретованістю. Результат: формалізовані правила оцінювання моделей.

6. Аналіз ризиків неправильного оцінювання моделі Необхідно вказати можливі ризики:
          - вибір невідповідної метрики;
          - переоцінка якості через leakage;
          - оцінювання лише на одній підвибірці;
          - ігнорування дисбалансу класів;
          - неврахування бізнес-вартості помилок;
          - орієнтація лише на технічні метрики без бізнес-аналізу. Для кожного ризику бажано подати короткий спосіб мінімізації. Результат: таблиця або список ризиків оцінювання.

  Очікувані результати та критерії оцінювання:
   - (1 бал) Тип задачі машинного навчання визначено коректно – вказано тип задачі, цільову змінну та очікуваний результат моделі.
   - (1 бал) Проаналізовано особливості даних – описано характеристики датасету, що впливають на вибір метрик.
   - (1 бал) Обрано основні метрики якості моделі – наведено релевантний перелік метрик і пояснено доцільність їх використання.
   - (1 бал) Обрано додаткові критерії оцінювання – враховано не лише точність, але й практичні властивості моделі.
   - (1 бал) Сформовано правила оцінювання та порівняння моделей – визначено основні та допоміжні метрики, спосіб валідації та критерії вибору найкращої моделі.
   - (1 бал) Виконано аналіз ризиків неправильного оцінювання – виявлено можливі помилки інтерпретації якості моделі та запропоновано способи їх уникнення.​ Разом: 6 балів
