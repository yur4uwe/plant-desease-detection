# Проектне завдання 15: Модульне тестування алгоритмів

      Мета виконання завдання: Навчитись застосовувати принципи модульного тестування до алгоритмів машинного навчання у Data Science. Набути навичок написання тестових сценаріїв, використання інструментів для тестування, автоматизації тестів та аналізу їх результатів на прикладах прикладних задач (класифікація, регресія, рекомендації, оптимізація, аналіз аномалій, текстовий аналіз).

План виконання:
1. Теоретичне ознайомлення
   - Ознайомтеся з поняттям модульного тестування:
          - Вивчіть різницю між unit, component, integration tests (ресурс).
          - Проаналізуйте, чому модульне тестування є критично важливим у розробці моделей машинного навчання.
   - Вивчіть методики перевірки моделей:
          - Ознайомтеся з train_test_split.
          - Ознайомтеся з крос-валідацією та її застосуванням (ресурс).

2. Розробка 2-х сценаріїв застосування моделей
   - Опишіть дві ситуації використання (наприклад, передбачення дефолту клієнта, прогноз врожайності тощо).
Один сценарій має містити, мінімум від 3 до 5 етапів.
   - Реалізувати відповідні Python-скрипти для:
          - Передбачення для нових даних.
          - Перевірки на фіксованому прикладі.

3. Розробка 2-х тестових сценаріїв для моделей
   - Напишіть unit-тести (з використанням unittest, pytest або doctest) для:
          - Перевірки коректності типів вихідних значень.
          - Обробки порожніх або “невалідних” даних.
          - Перевірки граничних умов.
   - Використайте assert, pytest.raises та фікстури (де потрібно).

4. Автоматизація тестування
   - Створіть тестовий пайплайн з:
          - Підготовкою даних.
          - Побудовою моделі.
          - Виконанням тестів.
   - Використайте pytest для запуску тестів.
   - Додати звітність: pytest-html або логування в консоль.

5. Проведення крос-валідації
   - Проведіть 5-кратну крос-валідацію (cross_val_score).
   - Порівняйте стабільність результатів (середнє значення, стандартне відхилення).
   - Візуалізуйте результати тестування.

Очікувані результати та критерії оцінювання:
   - (1 бал) Сценарії застосування моделі - два сценарії для використання розроблених конвеєрів.
   - (1 бал) Визначення граничних значень - граничні умови, в яких модель працює достовірно згідно визначених ймовірностей.
   - (1 бал) Тестові сценарії з pytest/unittest - два сценарії для тестування з описами.
   - (1 бал) Визначте покриття тестами реальних ситуацій - аналітично оцінене тестове покриття.
   - (1 бал) Скрипт для автоматизованого моніторингу - код програми для тестування.
   - (1 бал) Скріншоти візуалізацій - результати тестування моделей за допомогою крос-валідації.
